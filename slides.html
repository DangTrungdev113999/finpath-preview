Nhóm A — Trí nhớ AI

AI nhớ user hay sai ở đâu

Không chỉ watchlist — AI ghi nhớ hành vi, lỗi lặp, pattern ra quyết định.

AI ghi nhớ gì?

  • Watchlist + danh mục thực
  • Hành vi: bán vội, FOMO, gồng lỗ
  • Phản ứng với tin xấu (panic sell?)
  • Thời gian active (sáng/chiều/tối)
  • Lỗi lặp lại → nhắc nhở khi tương tự

Ví dụ thực tế trên thẻ

"Bạn cầm 2,000 HPG (35% danh mục).

Bạn đã 3 lần bán vội khi có tin xấu. Cả 3 lần giá quay lại trong 2-3 tuần."

→ Câu này từ Memory Layer, không phải data thị trường

Nhóm A — Việc cần làm

Chi + Hưởng: User Context

Onboarding thu thập profile

3-5 bước → xác định: thị trường, level (Retail/Pro), mã quan tâm, giờ active. Tối đa 60 giây.

Memory schema

Thiết kế cách lưu: watchlist, danh mục, hành vi, lỗi lặp. DB schema + API ghi/đọc.

Behavior detection

Rule đơn giản phát hiện pattern: bán trong 1h sau tin xấu = "panic sell", mua sau khi giá tăng 3 phiên liên tiếp = "FOMO".

Prompt cá nhân hóa

1 prompt duy nhất: nhận profile + memory → viết câu context ≤130 ký tự cho thẻ tín hiệu.

Chấm điểm tín hiệu

Không phải signal nào cũng quan trọng

Mỗi signal = 1-10 điểm. Chỉ ≥4 mới tạo thẻ.

Relevance — Liên quan đến user?

  • Trong danh mục → +4
  • Trong watchlist → +2
  • Cùng ngành → +1
  • Không liên quan → +0

Impact — Tác động mạnh cỡ nào?

  • Giá sập ≥7% → +4
  • Nhiều nguồn cùng chỉ → +3
  • 1 nguồn đáng tin → +2
  • KL tăng nhẹ → +1
Score = Relevance + Impact → ≥7 = nghiêm trọng (đỏ), 4-6 = quan trọng (vàng), <4 = bỏ qua
Gom nhóm tín hiệu

Compound & Adjacent

Compound — Nhiều nguồn → 1 mã

3 nguồn cùng chỉ về HPG:
• Tin tức: bị yêu cầu giải trình
• MXH: tiêu cực +340%
• Giá: –4.2%, vol 2.8x

→ Gộp thành 1 thẻ Compound, mạnh hơn từng signal riêng lẻ

Adjacent — Mã A → ảnh hưởng Mã B

DXG giảm sàn →
Nợ xấu BĐS tăng →
VCB (user đang cầm) bị ảnh hưởng

→ User không cầm DXG nhưng cần biết vì cầm VCB

Hưởng + Chi: Thiết kế logic gom nhóm. Compound = cùng mã + ≤4h. Adjacent = bản đồ liên quan giữa các mã/ngành.
Abnormal Return

Đo tác động thực — không gây hoảng loạn

Không có AR

"HPG bị điều tra → giảm 4.2%"

Gộp lẫn biến động thị trường. User hoảng.

→
Có AR

"HPG giảm thêm 2.45% so với thị trường"

Tách rõ: bao nhiêu do tin, bao nhiêu do TT chung.

Công thức:
AR = Actual Return – Expected Return
Expected = 0.5 × VNI Return + 0.5 × TB Ngành (loại trừ mã đó)

Confidence: |AR| ≥ 2% = Cao · 0.5–2% = TB · <0.5% = Thấp (loại)

Event Store

Lịch sử — "Lần trước như thế nào?"

Mỗi signal = 1 event lưu vào store. Lần sau tìm lại events tương tự → tra kết quả.

Tầng 1

Cùng mã + cùng loại
HPG bị điều tra → tìm HPG bị điều tra trước đó

Tầng 2

Cùng ngành + cùng loại
→ tìm HSG, NKG (Thép) bị điều tra

Tầng 3

Cùng tags + bất kỳ ngành
→ tìm FLC, DXG bị tạm giữ

Cold start: Seed 50-100 events lớn 24 tháng qua (1-2 ngày). Sau đó hệ thống tự tích lũy.

Tiêu đề ≤150 ký tự

Focal point duy nhất. User đọc mỗi cái này cũng đủ hiểu chuyện gì xảy ra.

Số liệu key ≤160 ký tự

Data cứng: giá, volume, % thay đổi. Đọc trong 3 giây.

Lịch sử ≤140 ký tự

Từ Event Store: "2 lần trước tương tự, hồi +12%". Subtle, không phải focal point.

Bối cảnh cá nhân ≤130 ký tự

Từ User Context: "Bạn cầm 2,000 HPG (35%)". AI biết user cầm gì.

Morning Briefing

Bản tin sáng 7:00 — độc bản cho mỗi user

Bao gồm gì?

  • Tóm tắt danh mục hôm qua (+/–%)
  • Mã cần chú ý hôm nay (tin, kỹ thuật)
  • Vĩ mô ảnh hưởng danh mục
  • Nhắc lỗi cũ nếu tình huống tương tự
  • Gợi ý: giữ / theo dõi / cân nhắc

Cá nhân hóa như thế nào?

  • Danh mục HPG 35% + VCB 42% → brief focus HPG, VCB
  • FPT trong watchlist → mention nếu có signal
  • Retail → "Đừng vội bán" · Pro → "AR –2.45%"
  • Hay bán vội → nhắc: "3 lần trước bạn bán, giá quay lại"
Thông báo thông minh

Push noti = prompt-based, không phải template

Cách cũ — Template cứng

"HPG có tin mới. Xem ngay!"

Mọi user nhận giống nhau. Spam. Bị tắt noti.

→
Cách mới — Context-aware

"HPG bạn cầm 35% vừa giảm 4.2%. 2 lần trước tương tự, giá hồi."

Riêng từng user. Chỉ gửi khi đáng gửi. Active cao.

Hưởng: Viết prompt cho push noti — nhận signal + user context → output noti text ≤100 ký tự. Anti-spam: tối đa 5 noti/ngày, ≥7 điểm mới push.
